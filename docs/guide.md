# 新手教程：从零到第一条隧道

> 如果你已经很熟悉 SSH 端口转发，这篇可以跳过，直接看 [README 的 Quick Start](../README.md#quick-start)。

## SSH 隧道是什么？

想象你的服务器上跑着一个数据库，端口 5432，但出于安全它只允许服务器自己访问。你在自己电脑上连不上它——除非"借道"SSH：让 SSH 在你电脑上开一个端口（比如也叫 5432），所有发到这个端口的数据都通过加密的 SSH 连接转交给服务器上的数据库。这条"借道"就是 SSH 隧道。

对你电脑上的软件来说，数据库就像跑在本地一样，连 `localhost:5432` 即可。

## 第一步：添加主机

主机就是你要连的那台服务器。打开 **Hosts** 页，点"添加"：

| 字段 | 填什么 | 例子 |
|---|---|---|
| 别名 | 随便起个好记的名字 | `我的云服务器` |
| 地址 | 服务器 IP 或域名 | `192.168.1.100` 或 `example.com` |
| 端口 | SSH 端口，没改过就是 22 | `22` |
| 用户名 | 登录服务器用的用户名 | `root` |
| 密钥文件 | 私钥路径，平时用 `ssh` 免密登录的话通常是 `~/.ssh/id_rsa`，留空则用 SSH 默认 | `C:\Users\你\.ssh\id_rsa` |
| 跳板机 | 需要先连一台中转机才能到目标机时选它，否则留空 | — |

保存后点 **测试**，显示绿色和延迟毫秒数就说明能连上。

> **注意**：Tunelo 调用系统的 `ssh` 命令，不会替你输密码，所以请先配置好密钥免密登录。如果你在终端里敲 `ssh user@host` 能不输密码直接进去，Tunelo 就能用。

已经有配好的 `~/.ssh/config`？直接点 **导入**，里面的 Host（连同 `LocalForward` 等转发规则）会被批量读进来，不用一台台手填。

## 第二步：创建隧道

打开 **Tunnels** 页，点"新建"，先选类型——这是唯一需要动脑子的地方：

**本地转发（L）—— 最常用**：把服务器上的服务"搬到"你电脑上。

> 例子：访问服务器上只对内开放的数据库。
> 本地端口填 `5432`，远程地址填 `localhost`，远程端口填 `5432`。
> 启动后，你电脑上连 `localhost:5432` = 连服务器上的数据库。
> （远程地址填 `localhost` 指的是"服务器自己"；也可以填服务器能访问到的其他机器的内网 IP。）

**远程转发（R）—— 反向**：把你电脑上的服务暴露到服务器上。

> 例子：本地开发的网站跑在 `localhost:3000`，想让服务器（或能访问服务器的人）看到。
> 启动后，在服务器上访问 `localhost:3000` = 访问你电脑上的网站。

**动态代理（D）—— SOCKS 代理**：在本地开一个 SOCKS5 代理端口，走这个代理的所有流量都经服务器转发出去。

> 例子：本地端口填 `1080`，然后在浏览器里设置 SOCKS5 代理 `127.0.0.1:1080`，浏览器的流量就全部从服务器出去了，相当于简易 VPN。

选好类型、填好端口、选择第一步添加的主机，保存。

## 常见场景：照着填就行

下面按界面上的实际字段给几个典型例子，找到和你需求最像的，改改端口就能用。

### 例 1：连服务器上的数据库（本地转发 L）

服务器上跑着 PostgreSQL（端口 5432），只监听 `127.0.0.1`，你想用本机的图形客户端连它。

| 字段 | 填 | 说明 |
|---|---|---|
| 名称 | `生产库` | 随意 |
| 转发类型 | `本地` | |
| 经过主机 | 选你的服务器 | |
| 本地 | `仅本机` + 端口 `5432` | 你电脑上开的端口 |
| 远端地址 | `localhost` | 指"服务器自己" |
| 远端端口 | `5432` | 数据库实际监听的端口 |

启动后，数据库客户端里填 `127.0.0.1:5432` 连接即可。MySQL 同理，把两个端口都换成 `3306`。

> 本地端口和远端端口不必相同。比如本机 5432 已被占用，本地端口填 `15432`，客户端连 `127.0.0.1:15432` 就行。

### 例 2：访问内网里另一台机器的服务（本地转发 L）

你能 SSH 到服务器 A，但真正想访问的是和 A 同内网的机器 B（比如 `192.168.10.8` 上端口 8080 的管理后台），B 你直接够不着。

| 字段 | 填 | 说明 |
|---|---|---|
| 转发类型 | `本地` | |
| 经过主机 | 服务器 A | 流量借道它 |
| 本地 | `仅本机` + 端口 `8080` | |
| 远端地址 | `192.168.10.8` | **B 的内网 IP**，不是 localhost |
| 远端端口 | `8080` | |

启动后浏览器打开 `http://127.0.0.1:8080`，看到的就是 B 上的后台。关键点：**远端地址填的是"从服务器 A 的视角"能访问到的地址**。

### 例 3：把本地开发的网页给远端看（远程转发 R）

你本机跑着开发服务器 `localhost:3000`，想让服务器（或服务器上的同事）访问到。

| 字段 | 填 | 说明 |
|---|---|---|
| 转发类型 | `远程` | |
| 经过主机 | 选你的服务器 | |
| 本地服务 地址 | `localhost` | 你本机上服务的地址 |
| 本地服务 端口 | `3000` | |
| 远端对外端口 | `仅服务端本机` + `3000` | 在服务器上开的端口 |

启动后，在**服务器上**访问 `http://127.0.0.1:3000` 就是你本机的网页。

> 想让**其他人**通过服务器的公网 IP 访问（`http://服务器IP:3000`），把"远端对外端口"改成 `全部网络`——但这需要服务器的 `sshd_config` 里配置 `GatewayPorts yes`，否则依然只有服务器本机能访问（界面上也有这条提示）。

### 例 4：让服务器用你本地的代理上网（远程转发 R）

你本机跑着代理软件（如 Clash，混合端口 `7890`），服务器网络受限（拉不动 GitHub、装不了包），想让服务器"借"你本地的代理出网。

| 字段 | 填 | 说明 |
|---|---|---|
| 转发类型 | `远程` | |
| 经过主机 | 选你的服务器 | |
| 本地服务 地址 | `localhost` | 代理跑在你本机上 |
| 本地服务 端口 | `7890` | 代理软件的监听端口 |
| 远端对外端口 | `仅服务端本机` + `7890` | 在服务器上开同名端口 |

启动后，在**服务器上**设置代理指向 `127.0.0.1:7890` 即可，例如：

```sh
export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890
curl -I https://github.com   # 走你本机的代理出网
```

### 例 5：浏览器走服务器上网（SOCKS5 代理 D）

想让浏览器"借"服务器的网络，访问只有服务器能访问的资源。

| 字段 | 填 | 说明 |
|---|---|---|
| 转发类型 | `SOCKS5` | |
| 经过主机 | 选你的服务器 | |
| 本地 | `仅本机` + 端口 `1080` | 代理监听的端口 |

启动后，在浏览器（或系统）的代理设置里填 **SOCKS5 代理 `127.0.0.1:1080`**，之后浏览器的所有流量都从服务器出去。D 类型不需要填远端地址——去哪由浏览器动态决定，这就是"动态"的意思。

### 例 6：先过跳板机再到目标机

公司要求先登跳板机（bastion）才能 SSH 到内网服务器。在 **Hosts** 页：

1. 先把**跳板机**加为一台主机并测试通过
2. 再添加**目标服务器**，"跳板机"字段选第 1 步加的那台

之后建隧道时"经过主机"直接选目标服务器即可，Tunelo 会自动走 `ProxyJump` 链路，多级跳板同理（跳板机也可以再配跳板机）。

## 第三步：启动和日常使用

- 点隧道旁的 **启动**，状态变为绿色"已连接"即成功
- **保持连接** 默认开启：断线后自动重连，无需手动操作
- 常用隧道打开 **自动启动**，配合设置里的"开机自启"，以后开机就自动连好
- 关掉窗口不会退出程序，它缩到系统托盘继续工作；真想退出，右键托盘图标选退出

## 常见问题

- **状态一直"重连中"** → 先回 Hosts 页测试主机能不能连上；再检查是不是需要密码（要配密钥免密）
- **提示端口被占用** → 本地端口被其他程序用了，换一个（1024–65535 之间随意）
- **找不到 ssh** → 去 Settings 页手动指定 ssh 的路径（Windows 一般在 `C:\Windows\System32\OpenSSH\ssh.exe`）
- **绑定地址是什么** → 默认 `127.0.0.1`，只有你自己的电脑能用这条隧道；改成 `0.0.0.0` 后同局域网的其他设备也能用，注意安全
